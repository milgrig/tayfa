# Tayfa — Мультиагентная система оркестрации

## Что это

Tayfa — это фреймворк для управления командой AI-агентов, которые работают как виртуальные сотрудники. Каждый агент имеет свою роль, навыки и зону ответственности. Агенты координируются через систему задач и спринтов, общаются в структурированном формате и автоматически коммитят результаты в Git.

## Ключевая идея

AI-агенты работают по модели реальной команды разработки:
- **Boss** — руководитель проекта, создаёт задачи и назначает исполнителей
- **Analyst** — детализирует требования, пишет acceptance criteria
- **Developer** — реализует функционал
- **Tester** — проверяет качество

## Основные возможности

### 1. Управление командой агентов
- Создание новых агентов через HR-процесс
- Каждый агент имеет профиль, навыки (skills), системный промпт
- Агенты специализируются на разных ролях

### 2. Система задач и спринтов
- Все задачи привязаны к спринтам
- Три роли в каждой задаче: постановщик → разработчик → тестировщик
- Статусы: новая → в_работе → на_проверке → выполнена
- Зависимости между задачами

### 3. Структурированная коммуникация
- Общение по задачам в файлах `.tayfa/common/discussions/{task_id}.md`
- Обязательные форматы handoff между этапами
- Чёткие acceptance criteria и test cases

### 4. Автоматизация Git
- Автоматические коммиты при смене статуса задачи
- Ветки для спринтов (`sprint/S001`)
- Автоматический merge и теги при финализации

### 5. Оркестратор (Web UI)
- Визуальная доска задач (канбан по спринтам)
- Запуск агентов по кнопке
- Управление проектами

## Целевая аудитория

- Разработчики, использующие AI-ассистентов для автоматизации
- Команды, экспериментирующие с мультиагентными системами
- Исследователи AI-оркестрации

## Технологии

- **Backend**: Python, FastAPI
- **AI**: Claude Code CLI (Anthropic)
- **Хранение**: JSON-файлы (задачи, сотрудники)
- **VCS**: Git (автоматическая интеграция)
- **Frontend**: HTML/JS (статический)
